# db-concept

## API message structures

### Requests to Hub
```js
{
    type: 'request',
    op: "db | ...",
    id,
    // additional properties for said operation
}
```
Note: The properties from the hub to database object must also be present in the requests to hub except for the requestId which is generated by the hub to keep track of the request.

### Hub to Database
```js
{
    type: 'request',
    method: "GET | POST | PUT | PATCH | DELETE",
    endpoint: "Tasks | ...",
    opName: "CREATE_TASK | GET_TASK | ...",
    data,
    requestId, // Hub Generated ID
    id
}
```

### Database to Hub
```js
{
    type: 'handoff-response',
    requestId
}
```

### Hub to Responses
```js
{
    type: 'handoff-status',
    status: true | false
}
```

### Heartbeat Request
```js
{
    type: 'heartbeat',
}
```

### Heartbeat Response
```js
{
    type: 'heartbeat-response',
}
```
